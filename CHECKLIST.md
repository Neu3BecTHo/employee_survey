## 🧪 Сценарий 1: Создание опроса администратором

### 1.1 Вход как администратор
- [ ] Откройте http://localhost:8080/login
- [ ] Выберите "Иван Иванов" (admin)
- [ ] Проверьте, что отображаются admin-кнопки

### 1.2 Создание нового опроса
- [ ] Нажмите "Создать опрос"
- [ ] Введите название: "Тестовый опрос"
- [ ] Введите описание: "Это тестовый опрос для проверки"
- [ ] Нажмите "Создать"
- [ ] Проверьте, что опрос появился в списке со статусом "draft"

### 1.3 Добавление вопросов
- [ ] Нажмите "Редактировать" у созданного опроса
- [ ] Нажмите "Добавить вопрос"
- [ ] Введите текст вопроса: "Насколько вы довольны работой?"
- [ ] Выберите тип: "Выбор варианта" (OK)
- [ ] Укажите обязательность: "Да" (OK)
- [ ] Введите варианты: "Очень доволен,Доволен,Нейтрально,Недоволен"
- [ ] Проверьте, что вопрос добавился

### 1.4 Добавление текстового вопроса
- [ ] Нажмите "Добавить вопрос"
- [ ] Введите текст вопроса: "Что можно улучшить?"
- [ ] Выберите тип: "Текстовый" (Отмена)
- [ ] Укажите обязательность: "Нет" (Отмена)
- [ ] Проверьте, что вопрос добавился

### 1.5 Открытие опроса
- [ ] Измените статус опроса на "open"
- [ ] Нажмите "Сохранить изменения"
- [ ] Проверьте, что статус изменился на "open"

## 🧪 API тесты (через curl)

### 6.1 Базовые операции
```bash
# Получение пользователей
curl -H "X-User-Id: 1" http://localhost:8080/users

# Получение опросов  
curl -H "X-User-Id: 1" http://localhost:8080/surveys

# Создание опроса (admin)
curl -X POST -H "Content-Type: application/json" -H "X-User-Id: 1" \
  -d '{"title":"API Test","description":"Test"}' http://localhost:8080/surveys
```

### 6.2 Проверка ошибок
```bash
# Попытка создания опроса employee (должно быть 403)
curl -X POST -H "Content-Type: application/json" -H "X-User-Id: 2" \
  -d '{"title":"Test"}' http://localhost:8080/surveys

# Пустые ответы (должна быть ошибка валидации)
curl -X POST -H "Content-Type: application/json" -H "X-User-Id: 2" \
  -d '{"answers":[]}' http://localhost:8080/surveys/1/responses
```
